# 任务管理

统一管理项目任务，支持本地和飞书 Bitable 双向同步。

## 任务来源

### 本地任务 (TodoWrite)

适用于：
- 当前会话的临时任务
- 开发过程中的子任务
- 不需要团队协作的任务

### 飞书 Bitable 任务

适用于：
- 需要团队协作的任务
- 需要长期跟踪的任务
- 与产品研发流程关联的任务

## 核心操作

### 创建任务

当用户说"创建任务"时：

1. **收集任务信息**
   ```
   📝 创建新任务

   标题: [等待输入]
   描述: [等待输入]
   类型: [ ] 开发 [ ] 测试 [ ] 文档 [ ] 其他
   优先级: [ ] P0 [ ] P1 [ ] P2 [ ] P3
   负责人: [当前用户]
   截止日期: [等待输入]

   存储位置:
   [1] 本地 (TodoWrite)
   [2] 飞书 Bitable
   ```

2. **创建任务**

   本地任务：使用 TodoWrite 创建
   飞书任务：使用 `feishu_create_bitable_record` 创建

3. **确认创建**
   ```
   ✅ 任务已创建

   标题: 实现用户登录功能
   ID: TASK-001 (飞书) / todo-1 (本地)
   链接: [飞书链接] (如果是飞书任务)

   下一步建议:
   - 使用 /skill doc-workflow 创建技术方案
   - 分配给团队成员
   ```

### 查看任务

当用户说"查看任务"时：

```
📋 任务列表

本地任务 (3):
  #1 [ ] 实现用户登录功能 - P1
  #2 [ ] 编写单元测试 - P2
  #3 [✓] 更新文档 - P3

飞书任务 (5):
  TASK-001 [ ] 订单模块开发 - P0 - @张三 - 截止: 2026-03-15
  TASK-002 [ ] 支付模块开发 - P1 - @李四 - 截止: 2026-03-20
  TASK-003 [✓] 需求评审 - P2 - @王五
  TASK-004 [⏳] 技术方案编写 - P1 - @张三 (进行中)
  TASK-005 [ ] 测试用例编写 - P2 - @李四

统计:
  待处理: 5
  进行中: 1
  已完成: 1
```

### 更新任务

当用户说"更新任务"或"完成任务"时：

1. **选择任务**
   ```
   选择要更新的任务:

   [1] TASK-001 订单模块开发 (待处理)
   [2] TASK-004 技术方案编写 (进行中)
   [3] #1 实现用户登录功能 (待处理)
   ```

2. **选择操作**
   ```
   更新 TASK-001:

   [1] 开始任务 (待处理 → 进行中)
   [2] 完成任务 (待处理 → 已完成)
   [3] 更新进度
   [4] 添加评论
   [5] 修改截止日期
   [6] 转交给其他人
   ```

3. **执行更新**

   使用 `feishu_search_bitable` 查找任务
   使用 `feishu_create_bitable_record` 更新记录

### 任务筛选

支持按条件筛选任务：

```
用户: "查看我负责的任务"
用户: "查看 P0 优先级的任务"
用户: "查看本周到期的任务"
用户: "查看订单模块相关的任务"
```

## Bitable 任务表结构

默认任务表字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| task_id | 文本 | 任务 ID (TASK-XXX) |
| title | 文本 | 任务标题 |
| description | 多行文本 | 任务描述 |
| status | 单选 | 待处理/进行中/已完成/已取消 |
| priority | 单选 | P0/P1/P2/P3 |
| assignee | 人员 | 负责人 |
| due_date | 日期 | 截止日期 |
| module | 单选 | 所属模块 |
| related_doc | 超链接 | 关联文档 |
| created_at | 创建时间 | 创建时间 |
| updated_at | 修改时间 | 更新时间 |

## 任务与文档关联

任务可以关联到项目文档：

```
用户: "为 PRD 创建开发任务"

📋 将为 PRD 创建任务:

源文档: docs/product/prd-order.md
建议任务:
  1. 实现订单创建功能
  2. 实现订单查询功能
  3. 实现订单取消功能

是否创建？[全部] [选择] [取消]
```

创建的任务自动关联到源文档。

## 同步规则

本地任务和飞书任务的同步：

| 操作 | 本地 → 飞书 | 飞书 → 本地 |
|------|------------|------------|
| 创建 | 手动选择 | 自动导入 |
| 更新 | 手动同步 | 会话启动时同步 |
| 删除 | 不同步 | 不同步 |

## 错误处理

| 错误 | 处理 |
|------|------|
| Bitable 表不存在 | 提示创建或检查配置 |
| 权限不足 | 提示检查应用权限 |
| 任务 ID 冲突 | 自动重新分配 ID |
