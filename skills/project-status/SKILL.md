# 项目状态汇总

生成项目进度报告、周报、团队工作量统计。

## 核心功能

### 1. 周报生成

当用户说"生成周报"时：

1. **收集数据源**
   - Git 提交记录
   - 任务完成情况（飞书 Bitable）
   - 文档状态变更
   - 代码评审记录

2. **生成周报**
   ```
   📊 周报 (2026-03-03 ~ 2026-03-09)

   ## 本周完成

   ### 功能开发
   - ✅ 订单模块 PRD 编写 (docs/product/prd-order.md)
   - ✅ 订单模块技术方案 (docs/tech/tech-order.md)
   - ✅ 订单创建接口开发

   ### Bug 修复
   - 🐛 修复支付超时问题 (#123)
   - 🐛 修复库存扣减并发问题 (#124)

   ### 代码评审
   - 📝 评审 5 个 PR，提出 12 条建议

   ## 代码统计

   | 指标 | 数值 |
   |------|------|
   | 提交数 | 23 |
   | 新增行数 | +1,234 |
   | 删除行数 | -456 |
   | 文件变更 | 34 |

   ## 任务进度

   | 任务 | 状态 | 进度 |
   |------|------|------|
   | 订单模块开发 | 进行中 | 60% |
   | 支付模块开发 | 待开始 | 0% |
   | 测试用例编写 | 进行中 | 30% |

   ## 下周计划

   1. 完成订单模块开发
   2. 开始支付模块技术方案
   3. 补充单元测试

   ## 风险与阻塞

   ⚠️ 支付网关接口文档缺失，需要协调
   ```

3. **导出选项**
   ```
   周报已生成！

   导出选项:
   [1] 复制为 Markdown
   [2] 同步到飞书
   [3] 发送到飞书群
   [4] 保存为文件 (docs/meetings/weekly-20260309.md)
   ```

## 数据源

### Git 数据

使用 `git log` 获取：
- 提交数量和作者
- 代码行数变更
- 文件变更统计

```bash
# 本周提交
git log --since="1 week ago" --oneline

# 代码统计
git log --since="1 week ago" --shortstat
```

### 任务数据

从飞书 Bitable 获取：
- 任务状态分布
- 任务完成数量
- 成员任务分配

### 文档数据

从 `.claude/context/doc-index.md` 获取：
- 文档状态分布
- 文档数量统计

## 报告格式

### Markdown 格式

标准 Markdown，支持：
- 标题层级
- 表格
- 列表
- 代码块
- 链接

### 飞书格式

同步到飞书时自动转换：
- 表格 → 飞书表格
- 代码块 → 代码块
- 链接 → 超链接

## 自定义配置

在 `.claude/context/report-config.md` 中配置：

```yaml
report:
  # 周报时间范围
  week_start: monday  # monday | sunday

  # 包含的数据源
  sources:
    git: true
    tasks: true
    documents: true

  # 排除的路径
  exclude_paths:
    - "docs/drafts/"
    - "**/*.test.js"

  # 自定义模板
  template: default  # default | minimal | detailed
```

## 定时报告（可选）

支持自动生成并发送报告：

```yaml
schedule:
  # 每周五 18:00 生成周报
  weekly_report:
    enabled: true
    time: "Friday 18:00"
    channels:
      - feishu_group: "OC-XXXX"
```

## 错误处理

| 错误 | 处理 |
|------|------|
| Git 仓库不存在 | 跳过 Git 数据，使用其他数据源 |
| 飞书连接失败 | 仅生成本地报告 |
| 无任务数据 | 显示 "暂无任务数据" |
| 时间范围无效 | 使用默认范围（本周） |
