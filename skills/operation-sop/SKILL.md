# 运营项目流程 SOP

标准的运营项目流程，从分析到复盘的完整指南。

## 流程概览

```
数据分析 → 策略制定 → SOP执行 → 效果评估 → 复盘总结
    ↓           ↓           ↓           ↓           ↓
 分析报告     策略文档    执行记录    评估报告    复盘报告
```

## 阶段 1: 数据分析

### 触发条件

- 开始新的数据分析任务
- 用户说"开始数据分析"
- 项目初始化，项目类型为 operation

### 执行步骤

1. **创建分析报告**
   ```
   用户: "分析上月的销售数据"

   AI 操作:
   - 使用 doc-workflow Skill 创建分析报告
   - 应用 templates/analysis.md 模版
   - 引导用户说明分析目标和数据来源
   ```

2. **数据收集**

   确定数据来源：
   - [ ] 飞书多维表格
   - [ ] 数据库查询
   - [ ] 第三方工具导出
   - [ ] 手动提供

3. **分析执行**

   进行数据分析：
   - [ ] 趋势分析
   - [ ] 对比分析
   - [ ] 漏斗分析
   - [ ] 分布分析

4. **结论输出**
   ```
   用户: "分析完成"

   AI 操作:
   - 更新文档状态为 approved
   - 建议下一步：制定策略
   ```

### 交付物

- 分析报告 (`docs/analysis/analysis-{period}-{topic}.md`)
- 状态: `approved`

### 退出条件

分析报告完成，进入策略制定阶段

---

## 阶段 2: 策略制定

### 触发条件

- 数据分析已完成
- 用户说"制定策略"、"提出优化方案"

### 执行步骤

1. **创建策略文档**
   ```
   用户: "基于分析结果制定提升策略"

   AI 操作:
   - 使用 doc-workflow Skill 创建策略文档
   - 应用 templates/strategy.md 模版
   - 引导用户填写策略内容
   ```

2. **策略内容**

   确保包含：
   - [ ] 背景与目标
   - [ ] 策略方案
   - [ ] 实施计划
   - [ ] 资源需求
   - [ ] 预期效果

3. **评审与确认**
   ```
   用户: "策略完成，提交评审"

   AI 操作:
   - 更新文档状态为 review
   - 提醒相关人员评审
   ```

4. **策略确认**
   ```
   用户: "策略已确认"

   AI 操作:
   - 更新文档状态为 approved
   - 建议下一步：执行策略
   ```

### 交付物

- 策略文档 (`docs/strategy/strategy-{period}-{topic}.md`)
- 状态: `approved`

### 退出条件

策略文档确认，进入执行阶段

---

## 阶段 3: SOP执行

### 触发条件

- 策略已确认
- 用户说"开始执行"、"执行策略"

### 执行步骤

1. **创建/更新 SOP**
   ```
   用户: "创建执行 SOP"

   AI 操作:
   - 使用 doc-workflow Skill 创建 SOP
   - 应用 templates/sop.md 模版
   - 记录执行步骤
   ```

2. **SOP 内容**

   确保包含：
   - [ ] 操作步骤
   - [ ] 责任人
   - [ ] 执行频率
   - [ ] 检查点
   - [ ] 异常处理

3. **执行追踪**
   ```
   使用 task-management Skill 追踪执行任务
   ```

4. **执行记录**
   ```
   每次执行后记录：
   - 执行时间
   - 执行人
   - 执行结果
   - 异常情况（如有）
   ```

### 交付物

- SOP 文档 (`docs/sop/sop-{topic}.md`)
- 执行记录（在 SOP 文档中或飞书 Bitable）

### 退出条件

策略执行一个周期后，进入效果评估阶段

---

## 阶段 4: 效果评估

### 触发条件

- SOP 执行一段时间后
- 用户说"评估效果"、"查看执行结果"

### 执行步骤

1. **创建评估报告**
   ```
   用户: "评估本季度执行效果"

   AI 操作:
   - 使用 doc-workflow Skill 创建评估报告
   - 收集 SOP 执行数据
   - 对比策略目标与实际效果
   ```

2. **评估内容**

   确保包含：
   - [ ] 目标回顾
   - [ ] 实际效果
   - [ ] 差异分析
   - [ ] 原因分析
   - [ ] 改进建议

3. **评估确认**
   ```
   用户: "评估完成"

   AI 操作:
   - 更新文档状态为 approved
   - 建议下一步：复盘总结
   ```

### 交付物

- 评估报告 (`docs/dashboard/evaluation-{period}.md`)
- 状态: `approved`

### 退出条件

效果评估完成，进入复盘阶段

---

## 阶段 5: 复盘总结

### 触发条件

- 效果评估已完成
- 用户说"复盘"、"总结经验"

### 执行步骤

1. **创建复盘报告**
   ```
   用户: "创建复盘报告"

   AI 操作:
   - 使用 doc-workflow Skill 创建复盘报告
   - 应用 templates/review.md 模版
   - 整合评估报告内容
   ```

2. **复盘内容**

   确保包含：
   - [ ] 整体回顾
   - [ ] 成功经验
   - [ ] 教训总结
   - [ ] 改进计划
   - [ ] 下一步计划

3. **复盘确认**
   ```
   用户: "复盘完成"

   AI 操作:
   - 更新文档状态为 archived
   - 标记项目阶段
   ```

### 交付物

- 复盘报告 (`docs/review/review-{period}.md`)
- 状态: `archived`

### 退出条件

项目复盘完成，可选择开启新周期

---

## 状态流转图

```
数据分析 → 策略制定 → SOP执行 → 效果评估 → 复盘总结
    ↓           ↓           ↓           ↓           ↓
  draft      draft       draft      draft      draft
    ↓           ↓           ↓           ↓           ↓
  review     review     (执行中)    review     archived
    ↓           ↓                       ↓
 approved   approved               approved
```

## 协作模式建议

| 阶段 | 推荐模式 | 说明 |
|------|----------|------|
| 数据分析 | Mode B | 人主导分析方向，AI 辅助处理 |
| 策略制定 | Mode A | 团队讨论为主，AI 记录整理 |
| SOP执行 | Mode D | AI 自动追踪和记录 |
| 效果评估 | Mode D | AI 自动生成报告 |
| 复盘总结 | Mode D | AI 自动汇总 |

---

## 触发下一步

当一个阶段完成时，AI 应该主动建议下一步：

```
✅ 数据分析完成

建议下一步:
[1] 制定优化策略 (使用 doc-workflow)
[2] 查看历史分析 (使用 project-status)
[3] 同步到飞书
```

---

## 周期性任务

运营项目通常是周期性的，AI 应该主动提醒周期性任务：

```
📅 周期性提醒:

本周任务:
- [ ] 周一: 执行数据同步 SOP
- [ ] 周三: 查看关键指标
- [ ] 周五: 生成周报

建议使用 task-management Skill 管理周期性任务。
```
